# Volume III: Repeated Games

## Purpose

This volume studies cooperation across repeated interaction.

It asks how reciprocal behavior can emerge, stabilize, degrade, or collapse over time when actors remember each other and expect future encounters.

## Core question

Can cryptographically remembered commitments improve long-term cooperation compared with simpler social norms, or do they create new failure modes?

## Scope

This volume covers repeated games, reciprocal strategy, delayed reciprocity, asymmetric reciprocity, trust networks, coalition formation, defection, punishment, forgiveness, and long-horizon fitness.

It does not assume that human cooperation is reducible to a payoff matrix.

## What this volume claims

- Cooperation changes when actors expect future interaction.
- Memory, identity continuity, and credible constraints can make behavior more legible across time.
- Reciprocity can be delayed, asymmetric, and still cooperative.
- Forgiveness can improve long-term fitness when it enables repair without erasing memory.
- Trust networks can become more robust through repeated evidence, but they can also cartelize.
- Cryptographic memory can improve accountability, but it can also harden conflict.

## What this volume does not claim

- It does not claim all relationships are games.
- It does not claim optimal strategy captures love, duty, family, honor, or sacrifice.
- It does not claim punishment always stabilizes cooperation.
- It does not claim more memory always produces better outcomes.
- It does not claim reciprocal commitments always outperform informal trust.
- It does not claim cryptographic enforcement prevents revenge spirals.

## Runtime relevance

This volume is relevant to runtime mechanisms that expose behavior across time:

- paid job history;
- signed receipts;
- attestations;
- reputation telemetry;
- local event chain health;
- requester proof paths;
- marketplace trust gates;
- future dispute and repair flows;
- future counterparty-specific reputation.

The current runtime provides evidence of events, but it does not yet model repeated counterparty relationships in full.

## Failure modes

- Cryptographic memory creates brittle escalation.
- Reputation telemetry becomes cartel infrastructure.
- Users optimize for visible proof rather than cooperative behavior.
- Forgiveness is absent, so all remembered failure becomes permanent punishment.
- Forgiveness exists but becomes laundering.
- Asymmetric reciprocity becomes dependency or extraction.
- Sybil actors reset identity faster than reputation can track behavior.

## Falsification focus

This volume fails wherever cryptographically remembered commitments produce brittle escalation, cartel behavior, performative cooperation, or lower long-term fitness than simpler social norms.

It also fails if HODLXXI increases visible accountability while decreasing actual willingness to cooperate.

## Open questions

- When does delayed reciprocity become exploitation?

- What observations would show that cryptographic commitments reduce long-term fitness?
- How should HODLXXI measure cooperation without reducing it to a naive score?
