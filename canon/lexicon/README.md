# CRT Canon Lexicon

The Lexicon defines the core language of Cryptographic Reciprocity Theory.

This is not a glossary of convenient words. It is the operational language used to connect theory, architecture, runtime mechanisms, observations, and falsification tests.

## Rule

A term is not canonical until it can support:

1. an operational definition;
2. observable behavior;
3. counterexamples;
4. implications for HODLXXI;
5. open questions or falsification paths.

## Working discipline

Theory defines language.

Language defines architecture.

Architecture defines code.

Code produces observations.

Observations revise theory.
