# Security Policy

## Reporting vulnerabilities
Please report suspected vulnerabilities privately to **security@hodlxxi.com**.
Do not open public issues for exploitable weaknesses.

Include:
- Affected component and version/commit.
- Reproduction steps.
- Potential impact.
- Suggested mitigation (if known).

## Scope
This repository includes research code and simulation scaffolding. Findings may relate to:
- Cryptographic protocol assumptions.
- Incentive/mechanism vulnerabilities.
- Implementation defects in simulation logic.
- Data handling and privacy leaks in proposed pipelines.

## Disclosure process
1. Acknowledge receipt within 5 business days.
2. Triage severity and affected artifacts.
3. Coordinate fix timeline.
4. Publish advisory after mitigation or documented risk acceptance.

## Safe harbor
Good-faith, non-destructive research is welcomed. Avoid actions that exfiltrate data,
degrade availability, or violate law/policy.
